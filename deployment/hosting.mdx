## Overview

After building your portfolio, you can deploy it to various hosting platforms. This guide covers the most popular options for hosting static React applications.

## Environment Variables

Your portfolio uses EmailJS for the contact form, which requires three environment variables:

```bash .env
VITE_SEVICE_ID=your_emailjs_service_id
VITE_TEMPLATE_ID=your_emailjs_template_id
VITE_PUBLIC_KEY=your_emailjs_public_key
```

<Note>
  In Vite, environment variables must be prefixed with `VITE_` to be exposed to your application. This is a security feature to prevent accidentally exposing server-side secrets.
</Note>

### Getting EmailJS Credentials

<Steps>
  <Step title="Create an EmailJS account">
    Sign up at [emailjs.com](https://www.emailjs.com/) if you haven't already.
  </Step>
  
  <Step title="Get your Public Key">
    Navigate to Account → API Keys to find your Public Key (`VITE_PUBLIC_KEY`).
  </Step>
  
  <Step title="Create an Email Service">
    Go to Email Services → Add New Service. Connect your email provider and note the Service ID (`VITE_SEVICE_ID`).
  </Step>
  
  <Step title="Create an Email Template">
    Go to Email Templates → Create New Template. Customize your template and note the Template ID (`VITE_TEMPLATE_ID`).
  </Step>
</Steps>

## Deployment Platforms

<Tabs>
  <Tab title="Vercel">
    ### Deploy to Vercel
    
    Vercel offers the simplest deployment experience with automatic deployments from Git.
    
    <Steps>
      <Step title="Install Vercel CLI (Optional)">
        ```bash
        npm install -g vercel
        ```
      </Step>
      
      <Step title="Deploy via CLI">
        From your project root:
        
        ```bash
        vercel
        ```
        
        Follow the prompts to link your project.
      </Step>
      
      <Step title="Or Deploy via Git Integration">
        1. Push your code to GitHub, GitLab, or Bitbucket
        2. Go to [vercel.com](https://vercel.com) and sign in
        3. Click "Add New Project"
        4. Import your repository
        5. Vercel auto-detects Vite configuration
        6. Click "Deploy"
      </Step>
      
      <Step title="Configure Environment Variables">
        In your Vercel project dashboard:
        
        1. Go to Settings → Environment Variables
        2. Add each variable:
           - `VITE_SEVICE_ID`
           - `VITE_TEMPLATE_ID`
           - `VITE_PUBLIC_KEY`
        3. Select all environments (Production, Preview, Development)
        4. Save and redeploy
      </Step>
    </Steps>
    
    #### Vercel Configuration (Optional)
    
    Create `vercel.json` for custom configuration:
    
    ```json vercel.json
    {
      "buildCommand": "npm run build",
      "outputDirectory": "dist",
      "framework": "vite"
    }
    ```
    
    <Note>
      Vercel provides automatic HTTPS, CDN distribution, and instant rollbacks. Each Git push creates a preview deployment.
    </Note>
  </Tab>
  
  <Tab title="Netlify">
    ### Deploy to Netlify
    
    Netlify offers continuous deployment, serverless functions, and form handling.
    
    <Steps>
      <Step title="Deploy via Drag & Drop">
        Quick deployment for testing:
        
        1. Run `npm run build` locally
        2. Go to [app.netlify.com](https://app.netlify.com)
        3. Drag the `dist` folder to the deploy zone
      </Step>
      
      <Step title="Or Deploy via Git Integration">
        For automatic deployments:
        
        1. Push your code to GitHub, GitLab, or Bitbucket
        2. Go to [app.netlify.com](https://app.netlify.com) and sign in
        3. Click "Add new site" → "Import an existing project"
        4. Connect your repository
        5. Configure build settings:
           - **Build command**: `npm run build`
           - **Publish directory**: `dist`
        6. Click "Deploy site"
      </Step>
      
      <Step title="Configure Environment Variables">
        In your Netlify site dashboard:
        
        1. Go to Site settings → Environment variables
        2. Click "Add a variable"
        3. Add each variable:
           - `VITE_SEVICE_ID`
           - `VITE_TEMPLATE_ID`
           - `VITE_PUBLIC_KEY`
        4. Save and trigger a new deploy
      </Step>
    </Steps>
    
    #### Netlify Configuration
    
    Create `netlify.toml` for advanced configuration:
    
    ```toml netlify.toml
    [build]
      command = "npm run build"
      publish = "dist"
    
    [[redirects]]
      from = "/*"
      to = "/index.html"
      status = 200
    ```
    
    <Warning>
      The redirect rule is crucial for React Router to work correctly. Without it, direct navigation to routes will return 404 errors.
    </Warning>
  </Tab>
  
  <Tab title="GitHub Pages">
    ### Deploy to GitHub Pages
    
    GitHub Pages offers free hosting for static sites directly from your repository.
    
    <Steps>
      <Step title="Install gh-pages package">
        ```bash
        npm install --save-dev gh-pages
        ```
      </Step>
      
      <Step title="Update vite.config.js">
        Set the base URL to your repository name:
        
        ```javascript vite.config.js
        export default defineConfig({
          base: '/your-repo-name/',
          // ... other config
        })
        ```
      </Step>
      
      <Step title="Add deployment scripts">
        Update `package.json`:
        
        ```json package.json
        {
          "scripts": {
            "dev": "vite",
            "build": "vite build",
            "preview": "vite preview",
            "predeploy": "npm run build",
            "deploy": "gh-pages -d dist"
          }
        }
        ```
      </Step>
      
      <Step title="Deploy">
        ```bash
        npm run deploy
        ```
        
        This builds your project and pushes the `dist` folder to the `gh-pages` branch.
      </Step>
      
      <Step title="Enable GitHub Pages">
        1. Go to your repository on GitHub
        2. Navigate to Settings → Pages
        3. Under "Source", select the `gh-pages` branch
        4. Click "Save"
        
        Your site will be available at `https://username.github.io/your-repo-name/`
      </Step>
    </Steps>
    
    #### Environment Variables on GitHub Pages
    
    <Warning>
      GitHub Pages doesn't support server-side environment variables. You have two options:
    </Warning>
    
    **Option 1: Use GitHub Actions** (Recommended)
    
    Create `.github/workflows/deploy.yml`:
    
    ```yaml .github/workflows/deploy.yml
    name: Deploy to GitHub Pages
    
    on:
      push:
        branches: [ main ]
      workflow_dispatch:
    
    permissions:
      contents: read
      pages: write
      id-token: write
    
    jobs:
      build:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          
          - name: Setup Node
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'
          
          - name: Install dependencies
            run: npm ci
          
          - name: Build
            env:
              VITE_SEVICE_ID: ${{ secrets.VITE_SEVICE_ID }}
              VITE_TEMPLATE_ID: ${{ secrets.VITE_TEMPLATE_ID }}
              VITE_PUBLIC_KEY: ${{ secrets.VITE_PUBLIC_KEY }}
            run: npm run build
          
          - name: Upload artifact
            uses: actions/upload-pages-artifact@v3
            with:
              path: './dist'
      
      deploy:
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}
        runs-on: ubuntu-latest
        needs: build
        steps:
          - name: Deploy to GitHub Pages
            id: deployment
            uses: actions/deploy-pages@v4
    ```
    
    Then add secrets in Settings → Secrets and variables → Actions.
    
    **Option 2: Hardcode values** (Not recommended for sensitive data)
    
    Create `.env.production` and commit it (only if values are public):
    
    ```bash .env.production
    VITE_SEVICE_ID=your_service_id
    VITE_TEMPLATE_ID=your_template_id
    VITE_PUBLIC_KEY=your_public_key
    ```
  </Tab>
  
  <Tab title="Cloudflare Pages">
    ### Deploy to Cloudflare Pages
    
    Cloudflare Pages offers fast global CDN and unlimited bandwidth.
    
    <Steps>
      <Step title="Connect your repository">
        1. Go to [dash.cloudflare.com](https://dash.cloudflare.com)
        2. Navigate to Pages
        3. Click "Create a project" → "Connect to Git"
        4. Select your repository
      </Step>
      
      <Step title="Configure build settings">
        - **Framework preset**: Vite
        - **Build command**: `npm run build`
        - **Build output directory**: `dist`
      </Step>
      
      <Step title="Add environment variables">
        1. Go to Settings → Environment variables
        2. Add each variable for Production:
           - `VITE_SEVICE_ID`
           - `VITE_TEMPLATE_ID`
           - `VITE_PUBLIC_KEY`
      </Step>
      
      <Step title="Deploy">
        Click "Save and Deploy". Your site will be live on `your-project.pages.dev`
      </Step>
    </Steps>
    
    <Note>
      Cloudflare Pages automatically handles SPA routing for React Router. No additional configuration needed.
    </Note>
  </Tab>
</Tabs>

## Custom Domain Setup

All platforms support custom domains:

<Steps>
  <Step title="Purchase a domain">
    Buy a domain from providers like Namecheap, Google Domains, or Cloudflare.
  </Step>
  
  <Step title="Configure DNS">
    Add DNS records pointing to your hosting platform:
    
    **For Vercel/Netlify/Cloudflare:**
    - Add CNAME record: `www` → `your-site.platform.com`
    - Add A record for apex domain (provided by platform)
  </Step>
  
  <Step title="Add domain in platform settings">
    - **Vercel**: Settings → Domains
    - **Netlify**: Site settings → Domain management
    - **GitHub Pages**: Settings → Pages → Custom domain
    - **Cloudflare Pages**: Custom domains → Set up a custom domain
  </Step>
  
  <Step title="Wait for DNS propagation">
    DNS changes can take 24-48 hours to propagate globally.
  </Step>
</Steps>

## Common Deployment Issues

### Issue: Blank Page After Deployment

**Possible Causes:**
- Incorrect `base` configuration in `vite.config.js`
- JavaScript errors (check browser console)
- Missing environment variables

**Solution:**
```javascript vite.config.js
// For root domain deployment
export default defineConfig({
  base: '/',
})

// For subdirectory deployment
export default defineConfig({
  base: '/subdirectory/',
})
```

### Issue: 404 on Page Refresh

**Problem**: Refreshing any route except home shows 404

**Solution**: Configure your hosting platform to redirect all routes to `index.html`:

<CodeGroup>
```toml Netlify
[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

```json Vercel
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

```yaml Cloudflare Pages
# Automatic - no configuration needed
```
</CodeGroup>

### Issue: Environment Variables Not Working

**Checklist:**
- [ ] Variables are prefixed with `VITE_`
- [ ] Variables are set in hosting platform dashboard
- [ ] Redeployed after adding variables
- [ ] Using `import.meta.env.VITE_VAR_NAME` (not `process.env`)

### Issue: Contact Form Not Working

**Common Problems:**
1. **CORS errors**: Verify EmailJS domain is whitelisted
2. **Invalid credentials**: Double-check Service ID, Template ID, and Public Key
3. **Missing environment variables**: Ensure all three variables are set correctly

**Testing:**
```javascript
// Add console logs in Contact.jsx to debug
console.log('Service ID:', import.meta.env.VITE_SEVICE_ID);
console.log('Template ID:', import.meta.env.VITE_TEMPLATE_ID);
console.log('Public Key:', import.meta.env.VITE_PUBLIC_KEY);
```

<Warning>
  Remove debug console logs before production deployment to avoid exposing sensitive data in browser console.
</Warning>

## Performance Optimization

### Enable Compression

Most platforms enable gzip/brotli compression automatically. Verify in browser DevTools:
- Check `Content-Encoding` header in Network tab
- Should show `gzip` or `br`

### Configure Caching

Vite automatically adds content hashes to filenames for cache busting:
```
assets/index-a1b2c3d4.js
assets/index-e5f6g7h8.css
```

This allows aggressive caching of assets while ensuring updates are always fetched.

### Optimize Images

Before deployment:
1. Compress images using tools like TinyPNG or Squoosh
2. Use modern formats (WebP, AVIF) with fallbacks
3. Implement lazy loading for below-the-fold images

## Deployment Checklist

Before going live:

- [ ] Production build completes without errors
- [ ] All environment variables configured on hosting platform
- [ ] Contact form tested and working
- [ ] All routes accessible and no 404 errors
- [ ] Custom domain configured (if applicable)
- [ ] HTTPS enabled
- [ ] Responsive design tested on mobile/tablet/desktop
- [ ] Browser compatibility tested (Chrome, Firefox, Safari)
- [ ] Lighthouse score reviewed (aim for 90+ on Performance)
- [ ] Meta tags and SEO optimized
- [ ] Analytics tracking configured
- [ ] Favicon and app icons present

## Continuous Deployment

All recommended platforms support automatic deployments:

1. **Push to Git**: Commit and push changes to your repository
2. **Automatic Build**: Platform detects changes and triggers build
3. **Deploy**: New version goes live automatically
4. **Rollback**: Instantly revert to previous deployment if needed

<Note>
  Set up branch-based deployments: production branch for live site, staging branch for testing.
</Note>
